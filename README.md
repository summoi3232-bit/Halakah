# حلقة — تطبيق تسيير حلقة التحفيظ

تطبيق ويب (PWA) لتسيير حلقات تحفيظ القرآن الكريم، يعمل بالكامل من المتصفح ويحفظ بياناته على الجهاز (بدون خادم أو إنترنت بعد أول تحميل).

## المسارات الأربعة
- **مسار الحفظ**: قائمة السور والآيات (من/إلى)، تقييم الحفظ (جيد / متوسط / ضعيف يُعاد)، ومقدار الحفظ للدرس القادم.
- **مسار المراجعة**: مراجعة القريب ومراجعة البعيد، كل منهما بقائمة سور وآيات وتقييم مستقل.
- **التلاوة النظرية**: متن تحفة الأطفال أو الجزرية (الأبيات من/إلى) مع عنوان مذكرة التجويد.
- **مسار التلاوة**: قائمة السور والآيات، تقييم (جيدة/ضعيفة) مع تفصيل الملاحظات: نقص المخارج والصفات، اختلاس الحركات، نقص المدود، نقص الغنة، أحكام النون الساكنة والتنوين، أحكام الميم الساكنة.

## ميزات أخرى
حفظ تلقائي فوري، تصدير/استيراد نسخة احتياطية (JSON)، تقارير يومية/أسبوعية/شهرية قابلة للطباعة، إحصائيات، متابعة الحضور، وبطاقة تعريف لكل متعلم (الاسم، السن، المستوى الدراسي).

## 1) رفع المشروع على GitHub

```bash
cd quran-tracker
git init
git add .
git commit -m "أول إصدار: تطبيق حلقة"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO-NAME.git
git push -u origin main
```
> استبدل `USERNAME/REPO-NAME` باسم مستودعك على GitHub (أنشئه أولاً من github.com/new).

## 2) النشر على Netlify

**الطريقة الأسهل (بدون أوامر):**
1. ادخل إلى https://app.netlify.com وسجّل الدخول.
2. اختر **Add new site → Import an existing project**.
3. اربط حساب GitHub واختر المستودع الذي رفعته.
4. اترك إعداد **Publish directory** فارغاً أو ضع نقطة `.` (ملف `netlify.toml` مُعدّ مسبقاً).
5. اضغط **Deploy site**. سيعطيك Netlify رابطاً مباشراً (يمكنك تخصيصه لاحقاً من Site settings → Domain management).

**أو عبر سطر الأوامر (Netlify CLI):**
```bash
npm install -g netlify-cli
netlify login
netlify deploy --prod
```

## 3) تحويل التطبيق إلى APK (تطبيق أندرويد)

بما أن التطبيق مبني كـ PWA (يحتوي `manifest.json` و `sw.js`)، أسهل طريقتين:

### الطريقة الأولى: PWABuilder (لا تحتاج برمجة)
1. انشر الموقع أولاً على Netlify (خطوة 2) واحصل على الرابط، مثل `https://your-site.netlify.app`.
2. ادخل إلى https://www.pwabuilder.com وألصق الرابط.
3. اضغط **Start**، ثم من تبويب **Android** اختر **Generate Package**.
4. حمّل ملف الـ APK (أو Android App Bundle) الجاهز للتثبيت أو للنشر على متجر Google Play.

### الطريقة الثانية: Bubblewrap (لمزيد من التحكم)
```bash
npm install -g @bubblewrap/cli
bubblewrap init --manifest https://your-site.netlify.app/manifest.json
bubblewrap build
```
سينتج ملف `app-release-signed.apk` جاهزاً للتثبيت.

> ملاحظة: كِلا الأسلوبين يتطلبان أن يكون الموقع منشوراً فعلياً عبر HTTPS (Netlify يوفر ذلك تلقائياً) قبل توليد الـ APK.

## هيكل الملفات
```
quran-tracker/
├── index.html      واجهة التطبيق
├── style.css        التصميم
├── app.js            منطق التطبيق وقاعدة البيانات المحلية
├── manifest.json      إعدادات PWA
├── sw.js              دعم العمل دون اتصال
├── icon.svg / icon-192.png / icon-512.png    أيقونات التطبيق
└── netlify.toml       إعداد النشر
```

## أمن البيانات
جميع بيانات المتعلمين والجلسات تُخزَّن محلياً في متصفح/جهاز المستخدم (localStorage). يُنصح باستخدام زر **"تنزيل نسخة احتياطية"** بشكل دوري (من صفحة النسخ الاحتياطي داخل التطبيق) لحفظ نسخة خارجية من البيانات.
